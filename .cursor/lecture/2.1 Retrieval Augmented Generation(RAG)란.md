# **RAG(Retrieval Augmented Generation)**

1. LLM 어플리케이션 개발에서 중요한 개념
    1. LLM 등장 전 NLP(자연어 처리) 에서부터 사용됨
    2. 우리말로 하면 검색-증강 생성

---

# **단어 뜯어보기**

1. Retrieval
    1. 데이터를 가져오는 것
    2. 구체적으로는 "컴퓨터 시스템에 저장된 자료를 취득하는 것" 이라는 뜻도 있음
    3. 언어모델이 가지고 있지 않은 정보를 가져오는 것
        1. 언어모델이 아웃풋을 만드는 데 필요한 정보를 제공하는 것
        2. 언어모델이 "답변 생성"에 능숙하지만, 답변 생성을 위한 모든 정보를 가지고 있지는 않음
            1. 보안이 걸려있는 사내자료 등 > LLM 이 모르는 정보
            2. GPT4는 23년 10월까지의 정보를 가지고 있음
2. Augmented
    1. AR/VR에 사용되는 단어
    2. 마치 사실인 것처럼
    3. Retrieval된 데이터를 LLM에게 주면서, "마치 이 정보를 아는 것처럼"
3. Generation
    1. 생성
    2. 내가 가져온 데이터를 제공할테니, 이 정보를 아는 것처럼, "답변을 생성해라"

---

# **이제 우리의 고민은?**

1. 우리는 뭘 해야하는가
    1. 답변을 생성하는 것은 LLM의 역할
        1. Generative AI (생성형 AI)가 Generation을 할 것임
    2. 우리는 데이터를 잘 가져와서
    3. LLM에게 잘 전달해야 함
        1. 우리는 RAG에서 **Retrieval Augmented** 까지만 하면 됨
2. 데이터를 잘 가져오려면?
    1. 잘 전달하기 위해선, 일단 잘 저장해야 함
    2. 잘 저장하는 것이 매우 매우 어려움
        1. **"어떻게** 저장하는가"가 강의에서 제일 중요한 부분이라고 생각함
        2. 부딪히면서 깨달은 여러가지 잔기술들을 소개할 예정
3. 잘 전달하려면?
    1. 프롬프트를 잘 활용해야 함
    2. 문맥을 어떻게 제공할 것인가도 매우 중요
    3. 잘 가져오더라도 제대로 전달하지 못하면 LLM이 올바른 답변을 주지 못함
        1. LangChain 활용의 장점
            1. LangChain은 RAG/채팅/요약 등 각 요구 별 쓰기 좋은 프롬프트를 제공하고있음
